## /explore — 头脑风暴工作流

> 工作流: /explore → /task next → /task done → /compound（可从任意步骤开始，不需按顺序）

通过多轮问答梳理项目场景和关键技术，产出两份结构化文档。

### 什么时候用

开始新功能或新项目时，先用 /explore 梳理思路，再进入 /task 生成具体任务。

### 产出文件

1. **docs/workflow/scene-alignment.md** — 目标用户、核心场景、成功标准、边界约束
2. **docs/workflow/key-technologies.md** — 技术栈、核心依赖、架构模式、关键决策、风险

### 两个阶段

1. **场景对齐** — 确认做什么、为谁做、成功标准是什么、不做什么
2. **关键技术** — 确认怎么做、用什么技术、有哪些风险

每个阶段通过一问一答渐进深入，信息足够后生成对应文件。

不适用：项目已有明确的 spec 或 PRD，或只是修复一个小 bug

用法：`/explore 项目或功能描述`
